1. proposals Table
Yeh table naye proposals ka sara data (13 sections, unka text, pricing table, toggles, status) save karegi.

      Haan bilkul! Hamare system me ab essentially **do tareeke ke workflows (types)** se Invoices ban rahe hain, jo ki ek hi common table (`invoice`) ko use karte hain. 

Dono me main difference ye hai ki wo **bante kahan se hain** aur unka **data database me kahan store hota hai**:

### 1. Legacy Calculator Invoices (Puraana System)
- **Kahan se Bante Hain:** Jab aap directly "Calculator" module me jaate the, wahan manually Graphic/Ads services select karke direct ek Final Invoice generate karte the.
- **Database Storage:** Inka data bohot saari alag-alag tables me tukdo me save hota hai. Jaise graphic services ke liye `invoice_graphic`, ads ke liye `ads_campaign_details_invoice`, add-ons ke liye `additional_service_invoice`, discounts ke liye `invoice_discount` aadi.
- **Flow:** Direct Invoice Generation -> Record Payments.

### 2. Proposal-Based Invoices (Naya System - Jispe abhi humne kaam kiya)
- **Kahan se Bante Hain:** Ye naye "Proposals" flow se bante hain. Pehle ek Proposal banta hai, usse ek **Proforma Invoice** generate hota hai (jo ki final invoice nahi hota, sirf advance demand ke liye hota hai).
- **Final Invoice Kab Banta Hai:** Jab aap uss Proforma par pehla **Payment Record** karte hain, tab system automatically usko ek **Final Invoice** me convert kar deta hai (aur use `invoice` table me daal deta hai `invoice_source = 'proposal'` column ke sath).
- **Database Storage:** Inme alag-alag (graphic/ads) tables me data save nahi hota. Balki poori services, notes, aur T&C ek single JSON column `pricing_snapshot` me hi save ho jaati hai (jo ki zyada fast aur clean tarika hai).
- **Flow:** Proposal -> Proforma Invoice -> Payment Received -> Automatically Final Invoice Generated.

**Summary:** 
Dono final invoices dikhte bilkul same hain PDF me (`AdminInvoice.jsx` dono ko handle karta hai), lekin ek directly Calculator se banta hai (Legacy), aur doosra ek structured step-by-step process (Proposal -> Proforma -> Payment) ke baad banta hai (New System).
